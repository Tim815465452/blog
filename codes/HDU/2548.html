HDU 2548

2017-05-13 09:21 by zsz12251665

Problem

vjudge
HDU

Solution

This problem is just a mathmatical problem. The speed how fast the two armies get close is (u+v), the sum of their speeds. Then the time they meet will be l/(u+v). The reporter is always running, so the time he runs is also l/(u+v). And the distance he runs will equals the speed multiply the time that is l/(u+v)*w. So when you read a group of (u,v,w,l), just output l/(u+v)*w.

Code

Pascal

var
    t,i:integer;
    u,v,w,l:double;
begin
    read(t);
    for i:=1 to t do
        begin
            read(u,v,w,l);
            writeln(l/(u+v)*w:0:3);//In Pascal, the programmer should pay attention that the / operator will give a real number answer, the div operator will give a integer answer. But the div operator can only calculate integers but not real numbers
        end;
end.

C++

#include<cstdio>
int main()
{
    int t;
    scanf("%d",&t);
    for(int i=0;i<t;i++)
    {
        double u,v,w,l;
        scanf("%lf%lf%lf%lf",&u,&v,&w,&l);
        printf("%.3lf\n",l/(u+v)*w);//In C/C++, the / operator will check the data type automatically. If both are integers it, it will give a integer answer(truncate the digits after the decimal point). Or It will give the real number answer
    }
    return 0;
}
